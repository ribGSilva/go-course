    Quickstart #

    Installation #

    To install Go, just follow the documentation in the official web page

    Is really simple to install.

    Version Manager #

    I personally like to use a version manager to easily change versions working cross projets.

    I use g, but there are a lot of others out there. Just pick the one you like more.

    Private Repository #

    If you want to use some private repository (and probably you will), you will need to configure 2 things:

    1 - GOPRIVATE #

    First, you will need to configure the env var GOPRIVATE.

    This variable will tell to go that those hosts listed are private repositories, example:

    GOPRIVATE=bitbucket.my.domain.com

    You can use multiples domains, just separate them using comma “,”

    GOPRIVATE=bitbucket.my.domain.com,bitbucket.org/my_private_repository

    2 - Configure the creadentials #

    Go handles its dependencies accessing directly the source code in the repositories. It uses http to download the sources.

    As you code is in a private repository, you will need to tell go to use git to do that.

    You can do that typing something like that:

    git config --global url."git@bitbucket.org:".insteadOf "https://bitbucket.org/"

    If you will use ssh, you can type:

    git config --global url."ssh://git@bitbucket.org:".insteadOf "https://bitbucket.org/"
