    Basics #

    So let’s create a basic addition code.

    package main

import "fmt"

func addInt(x int, y int) int {
    return x + y
}

func printResult(result string) {
    fmt.Println(result)
}

func main() {
    a := 5
    b := 3

    sum := addInt(a, b)
    messageResult := fmt.Sprintf("the sum of %d and %d is %d", a, b, sum)

    printResult(messageResult)
}

    Functions #

    Here we created our first function. It is composed by 3 main points:

    Function Name #

    Its name:

    addInt

    Function Params #

    Its params:

    (x int, y int)

    The params must always be between parentheses, and for params of same type, we can omit type of all but the last param.

    Like this:

    func exemple(x, y, z int) int {
    return x + y + z
}

    And we can blend types

    func addPrint(x, y, z int, messageFormat string) {
    fmt.Printf(messageFormat, x + y + z)
}

    Function Returns #

    Its return:

    int

    If the function returns something, we need to use “:” and put the type of return.

    We can return more then one value, but we need to put it between parentheses.

    func example(x, y int, messageFormat string) (int, string) {
    sum := x + y
    messageFormatted := fmt.Sprintf(messageFormat, sum)

    return sum, messageFormatted
}

    We can also define names for the results and assign them in the code, but if we do it for one return value, we need to put in all of them.

    func example(x, yint, messageFormat string) (sum int, messageFormatted string) {
    sum = x + y
    messageFormatted = fmt.Sprintf(messageFormat, sum)
    return
}

    The difference is that you don’t need to init the variables of return, they are already initialized, and you don’t need to pass them in the return statement.

    Variable Initialization #

    Talking about variable initialization, we have some different ways.

    We can use var to init variables.

    package main

import "fmt"

func main() {

    var var1 int
    var var2, var3 int
    var var4, var5 = 9, 5

    var3 = 4

    fmt.Println(var1) // 0
    fmt.Println(var2) // 0
    fmt.Println(var3) // 4
    fmt.Println(var4) // 9
    fmt.Println(var5) // 5
}

    As we can see, if we don’t pass any value, the var will be stated with default/empty value.

    We can also init variables with already with values not using var. We can use := , but in this case, we can’t init the variable with no value assigned.

    package main

import "fmt"

func main() {

    var1 := 3
    var2, var3 := 2, 4

    fmt.Println(var1) // 3
    fmt.Println(var2) // 2
    fmt.Println(var3) // 4
}

    If some variable is already created and you want to assign other value to it and create a new variable at the same time, you need to use the := notation.

    package main

import "fmt"

func main() {

    var1 := 3

    var1, var2 := functionWithTwoReturns()

    fmt.Println(var1)
    fmt.Println(var2)
}

    Switch Case #

    The switch case follows that model

    func main() {
	s := "key"
	switch s {
	case "key":
		println("value is key")
	case "wallet":
		println("value is wallet")
		fallthrough // thill will call the case below
	case "money":
		println("value is money")
	default:
		println("unmapped value")
	}
}

    If the expressions fits some case, it will execute and quit the switch.

    You can execute thecase and the case below using the command fallthrough.

    If the value does not fit any case, it will execute the default case.

    If there is no default case, it will do nothing.

    You can also create an empty switch case:

    func main() {
    myvar := 4
	switch {
	case 1 == myvar:
		println("value is one")
	case myvar > 5:
		println("value is more then five")
}

    Arrays/Slice #

    In go, we have arrays and slices. You don’t need to worry about the differences now, so let’s pretend that they are the same.

    The notaion is:

    []type

    Example:

    []string // string array
[]int    // int array

    Create #

    We can create a slice this ways:

    func main() {
    length := 10
    s := make([]string, length)

    var array [10]string
}

    In that case, we created a slice with 10 positions.

    We can specify the total capacity of the array, and the actual length of it.

    func main() {
    capacity := 10
    length := 2
    s := make([]string, length, capacity)
}

    We can also create a slice already with values.

    func main() {
	a := []int{5, 3, 4}
}

    Manipulate #

    We can manipulate the values using the index

    func main() {
	a := []int{5, 3, 4}
    println(a[1]) // 3
    a[1] = 9
    println(a[1]) // 9
}

    We can select only part of the arrays

    func main() {
	a := []int{5, 8, 4, 7}
	fmt.Println(a[0:2]) // [5 8]
    b := a[2:4]
    fmt.Println(b) // [4 7]
    fmt.Println(b[1]) // 7
}

    Map #

    Maps as alike slices, but they does not have total capacity

    The notaion is:

    map[keyType]valueType

    Example:

    map[int]string // map with int key, and string value
map[int]int    // map with int key, and int value

    Create #

    We can create a map like this

    func main() {
    m := make(map[int]string)
}

    We can also create a map already with values

    func main() {
	m := map[int]string{
		1: "value1",
		2: "value2",
	}
	fmt.Println(m)    // map[1:value1 2:value2]
	fmt.Println(m[1]) // value1
}

    Manipulate #

    As slices, we can manipulate the map using the index

    func main() {
	m := map[int]string{
		1: "value1",
	}
	fmt.Println(m)    // map[1:value1]
	fmt.Println(m[1]) // value1
    m[1] = "newValue"
    m[2] = "value"
	fmt.Println(m)    // map[1:newValue 2:value]
}

    Channel #

    Channels are alike queues, they can store data, and deliver data.

    The notaion is:

    chan type

    Example:

    chan string // channel of strings
chan int    // channel of ints

    Create # 

    func main() {
    capacity := 10
	c := make(chan int, capacity)
}

    Manipulate #

    We can add or remove data from channels with a special operation

    func main() {
	capacity := 10
	c := make(chan int, capacity)

	c <- 1
	c <- 2
	c <- 3

	v1 := <-c
	v2 := <-c
	v3 := <-c

	println(v1) // 1
	println(v2) // 2
	println(v3) // 3
}

    If the channel is full, it will block the current thread until there is space to store

    func main() {
	capacity := 2
	c := make(chan int, capacity)

	c <- 1
	c <- 2
	c <- 3 // this will block the thread execution
}

    We can also pass a second param to receive if the channel still working

    func main() {
	capacity := 2
	c := make(chan int, capacity)

	c <- 1

    v, ok := <- c
    if ok {
        println(v)
    } else {
        println("closed channel")
    }
}

    Loops #

    Go suports only one command of loop, the for statment.

    Indexed #

    We have the for statement that executes a loop

    func main(){
    for i := 0; i < 5; i++ {
        println(i)
    }
}

    Infinite #

    We can create infinit loops.

    func main(){
    i := 0
    for {
        println(i)
        if i > 5 {
            break;
        }
        i++
    }
}

    To interact with loops, we can use the commands: break, continue

    • break: exists the loop
    • continue: goes to the next iteration

    Iteration #

    We can also iterate through maps and slices/arrays.

    You will need to use the range command.

    Arrays/slice # 

    func main() {
	list := []string{"val1", "val2"}
	for i, v := range list {
		fmt.Printf("position %d has the value %s", i, v)
	}
}

    Maps # 

    func main() {
	m := map[int]string{
		1: "value1",
		2: "value2",
	}
	for k, v := range list {
		fmt.Printf("key %d has the value %s", k, v)
	}
}

    Embedded functions #

    In go we have some native funcs to help us to work with go.

    print #

    To print in the console, we have the basic functions

    func main() {
    print("someoutput")
    println("print and break line")
}

    make #

    To create arrays/slice, maps or channels, we can use the func “make”

    func main() {
    length := 4
    capacity := 10
    myArray := make([]int, length, capacity)

    myMap := make(map[string]int)

    myChan := make(chan int)
}

    len #

    We can check the length of channels, maps and arrays/slices using len()

    func main() {
    length := 4
    capacity := 10
    myArray := make([]int, length, capacity)
    println(len(myArray)) // 4

    myMap := make(map[string]int)
    println(len(myMap)) // 0

    capacityC := 5
    myChan := make(chan int, capacityC)
    myChan <- i // add data to the channel
    println(len(myChan)) // 1
}

    cap #

    We can check the total capacity of a channel or an array/slice using cap()

    func main() {
    length := 4
    capacity := 10
    myArray := make([]int, length, capacity)
    println(cap(myArray)) // 10

    capacityC := 5
    myChan := make(chan int, capacityC)
    myChan <- i // add data to the channel
    println(cap(myChan)) // 5
}

    append #

    We can append data to and array/slice using append()

    Append will expand the capacity of yout array if it does not have capacity to store the new value.

    func main() {
    myArray := make([]string, 0, 1)
    println(len(myArray)) // 0
    println(cap(myArray)) // 1
    println("---")

    myArray = append(myArray, "someString")
    println(len(myArray)) // 1
    println(cap(myArray)) // 1
    println("---")

    myArray = append(myArray, "moreData")
    println(len(myArray)) // 2
    println(cap(myArray)) // 2
}

    defer #

    Defer is command that tells go to execute that function at the end of the current function, no matter what happens, ether a return or a panic.

        type myStr struct {
        Data string
    }

    func main() {
        defer func() {
            println("main exited")
        }()

        var str *myStr
        println(str.Data)
    }

    This will result in something like this:

    ...
main exited
panic: runtime error: invalid memory address or nil pointer dereference
...

    new #

    The function new creates a new pointer of the type passed as param with empty value (not nil)

    func main() {
    str := new(string)
    println(*str) // ""
}

    close #

    The clonse function is used to close channels.

    func main() {
    c := make(chan int, 1)

    c <- 0

    <- c

    close(c)
}
