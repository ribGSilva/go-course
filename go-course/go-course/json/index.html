    JSON #

    Work With Json #

    To handle json, go has a native librabry for that.

    encoding/json

    It is really easy to use it.

    Check the example below.

    package main

import (
	"encoding/json"
	"fmt"
)

const myJson = `{
	"name": "my name",
	"age": 15,
	"alive": true,
	"extras": "extra infos",
	"privateField": "using a private field",
    "omitted": "field omitted if empty"
}`

type user struct {
	Name         string `json:"name"`
	Age          int    `json:"age"`
	Alive        bool   `json:"alive"`
	DateOfBirth  string `json:"dob"`
	privateField string `json:"privateField"`
	Omitted      string `json:"omitted,omitempty"`
}

func main() {

	var u user
	if err := json.Unmarshal([]byte(myJson), &u); err != nil {
		fmt.Printf("failure to parse json to user: %s \n", err)
	} else {
		fmt.Printf("my user is: %+v \n", u)
	}

	u.DateOfBirth = "2000/01/01"
	u.privateField = "setting private field"
	u.Omitted = ""

	if bytes, err := json.Marshal(u); err != nil {
		fmt.Printf("failure to parse user to json: %s \n", err)
	} else {
		fmt.Printf("my json is: %s \n", string(bytes))
	}
}

    Running this code, we have this output:

    my user is: {Name:my name Age:15 Alive:true DateOfBirth: privateField: Omitted:field omitted if empty} 
my json is: {"name":"my name","age":15,"alive":true,"dob":"2000/01/01"}

    As we can see, we have some particular behaviours.

    • The Unmarshal method receives a ponter to the object to be filled
    • It doesn’t break if not all fields are mapped in json nether in the struct (you can change that if you want)
    • Private fields are not mapped
    • You can remove fields if empty using the tag “omitempty”
